Smithers includes a built-in Linear integration that lets workflows read issues, update statuses, post comments, listen for webhooks, and give agents direct access to your Linear workspace through AI SDK tools.

## Setup

Set the `LINEAR_API_KEY` environment variable with a [Linear API key](https://linear.app/settings/api):

```bash
export LINEAR_API_KEY=lin_api_...
```

## Import

```ts
import {
  useLinear,
  linearTools,
  getLinearClient,
  startWebhookServer,
  useLinearWebhook,
  LinearWebhookListener,
} from "smithers-orchestrator";
// or from the dedicated subpath:
import { useLinear, linearTools, startWebhookServer } from "smithers-orchestrator/linear";
```

> **Subpath-only exports:** Some symbols are only available from `smithers-orchestrator/linear` and are *not* re-exported from the main `smithers-orchestrator` barrel:
>
> - `resetLinearClient` — clears the cached Linear SDK client (useful in tests)
> - Individual tool functions: `linearListIssues`, `linearGetIssue`, `linearUpdateIssue`, `linearAddComment`, `linearListTeams`
> - Types: `LinearProject`, `LinearIssueStatus`, `ListIssuesParams`

## useLinear

A hook that returns typed operations for interacting with Linear. All return values are plain serializable objects (not SDK lazy models), safe to pass between tasks.

```ts
const linear = useLinear();
```

### listIssues

Fetch issues with optional filters.

```ts
const issues = await linear.listIssues({
  teamId: "team-uuid",
  stateType: "unstarted",   // "unstarted" | "started" | "completed" | "cancelled"
  limit: 20,
  labels: ["bug"],
});
```

**Parameters:**

| Field | Type | Description |
|---|---|---|
| `teamId` | `string?` | Filter by team ID |
| `assigneeId` | `string?` | Filter by assignee ID |
| `stateType` | `string?` | Filter by state type |
| `limit` | `number?` | Max results (default 50) |
| `labels` | `string[]?` | Filter by label names |

**Returns:** `LinearIssue[]`

### getIssue

Fetch a single issue by ID or identifier.

```ts
const issue = await linear.getIssue("JJH-42");
```

### updateIssueState

Move an issue to a new state.

```ts
await linear.updateIssueState(issue.id, doneStateId);
```

### addComment

Post a markdown comment on an issue.

```ts
const commentId = await linear.addComment(issue.id, "## Done\nShipped in v1.2.0");
```

### listComments <span style={{opacity:0.6}}>(programmatic only)</span>

Fetch all comments on an issue.

```ts
const comments = await linear.listComments(issue.id);
```

> `listComments` is only available as a programmatic method on the `useLinear()` hook. There is no corresponding agent tool in `linearTools`.

### listTeams

List all teams in the workspace.

```ts
const teams = await linear.listTeams();
const team = teams.find(t => t.key === "ENG");
```

### listProjects <span style={{opacity:0.6}}>(programmatic only)</span>

List all projects in the workspace.

```ts
const projects = await linear.listProjects();
```

> `listProjects` is only available as a programmatic method on the `useLinear()` hook. There is no corresponding agent tool in `linearTools`.

### listIssueStatuses <span style={{opacity:0.6}}>(programmatic only)</span>

List all workflow states for a team.

```ts
const statuses = await linear.listIssueStatuses(team.id);
const inProgress = statuses.find(s => s.name === "In Progress");
```

> `listIssueStatuses` is only available as a programmatic method on the `useLinear()` hook. There is no corresponding agent tool in `linearTools`.

## linearTools

Pre-built AI SDK tools that agents can use to interact with Linear directly. Pass them to any agent's `tools` option.

```ts
import { ToolLoopAgent as Agent } from "ai";
import { anthropic } from "@ai-sdk/anthropic";
import { linearTools } from "smithers-orchestrator/linear";
import { read, grep } from "smithers-orchestrator/tools";

const triageAgent = new Agent({
  model: anthropic("claude-sonnet-4-20250514"),
  tools: { ...linearTools, read, grep },
  instructions: "You are a ticket triage agent. Read issues and classify them.",
});
```

### Available tools

| Tool | Description |
|---|---|
| `linearListIssues` | List issues with optional filters |
| `linearGetIssue` | Get a single issue by ID or identifier |
| `linearUpdateIssue` | Update an issue's state |
| `linearAddComment` | Add a comment to an issue |
| `linearListTeams` | List all workspace teams |

### Importing individual tools

If you only need a subset of Linear tools, you can import them individually from the subpath instead of using the bundled `linearTools` object:

```ts
import {
  linearListIssues,
  linearGetIssue,
  linearUpdateIssue,
  linearAddComment,
  linearListTeams,
} from "smithers-orchestrator/linear";

const agent = new Agent({
  model: anthropic("claude-sonnet-4-20250514"),
  tools: { linearListIssues, linearGetIssue, read, grep },
});
```

> These individual tool exports are only available from `smithers-orchestrator/linear`, not from the main `smithers-orchestrator` barrel.

## Types

All types are plain serializable objects:

```ts
import type {
  LinearIssue,
  LinearTeam,
  LinearProject,
  LinearComment,
  LinearIssueStatus,
  ListIssuesParams,
} from "smithers-orchestrator/linear";
```

### LinearIssue

```ts
type LinearIssue = {
  id: string;
  identifier: string;         // e.g. "ENG-42"
  title: string;
  description: string | null;
  priority: number;            // 0=None, 1=Urgent, 2=High, 3=Normal, 4=Low
  priorityLabel: string;
  state: { id: string; name: string; type: string } | null;
  assignee: { id: string; name: string; email: string } | null;
  labels: { id: string; name: string }[];
  project: { id: string; name: string } | null;
  createdAt: string;           // ISO-8601
  updatedAt: string;           // ISO-8601
  url: string;
};
```

## Example: Ticket Processing Workflow

A complete workflow that processes Linear tickets through research, implementation, and review stages:

```tsx
import { createSmithers, Sequence, Loop, Parallel } from "smithers-orchestrator";
import { ClaudeCodeAgent, CodexAgent } from "smithers-orchestrator";
import { useLinear } from "smithers-orchestrator/linear";
import { read, grep, bash } from "smithers-orchestrator/tools";
import { z } from "zod";

const researchSchema = z.object({
  summary: z.string(),
  relevantFiles: z.array(z.string()),
  complexity: z.enum(["low", "medium", "high"]),
});

const resultSchema = z.object({
  ticketId: z.string(),
  success: z.boolean(),
});

const { Workflow, Task, smithers, outputs } = createSmithers({
  research: researchSchema,
  result: resultSchema,
});

const researcher = new ClaudeCodeAgent({
  model: "claude-opus-4-20250514",
  systemPrompt: "Research the ticket and find relevant code.",
});

export default smithers((ctx) => (
  <Workflow name="ticket-processor">
    <Sequence>
      <Task id="research" output={outputs.research} agent={researcher}>
        {`Research ticket: ${ctx.input.title}\n${ctx.input.description}`}
      </Task>
      <Task id="update-linear" output={outputs.result}>
        {async () => {
          const linear = useLinear();
          await linear.addComment(ctx.input.issueId, "Research complete.");
          return { ticketId: ctx.input.ticketId, success: true };
        }}
      </Task>
    </Sequence>
  </Workflow>
));
```

## Webhooks

Smithers can listen for Linear webhook events to trigger workflows when issues are updated. Three APIs are available depending on your use case.

### startWebhookServer

The imperative API starts a local HTTP server and auto-registers a webhook with Linear. On shutdown it deregisters the webhook automatically.

```ts
import { startWebhookServer } from "smithers-orchestrator";

const server = await startWebhookServer({
  publicUrl: "https://my-tunnel.ngrok.io",  // must be reachable from Linear
  port: 3456,
  teamId: "team-uuid",          // optional: scope to one team
  resourceTypes: ["Issue"],     // default
  label: "smithers-webhook",    // display name in Linear settings
});

// Listen for issue events
server.handler.on("Issue", (payload) => {
  console.log(payload.action, payload.data.identifier);
});

// Graceful shutdown — deregisters webhook from Linear and stops server
await server.shutdown();
```

**Options:**

| Option | Type | Default | Description |
|---|---|---|---|
| `publicUrl` | `string` | *required* | Public URL that Linear will POST to |
| `port` | `number` | `3456` | Local server port |
| `teamId` | `string?` | — | Scope webhook to a specific team |
| `resourceTypes` | `string[]` | `["Issue"]` | Linear resource types to subscribe to |
| `label` | `string` | `"smithers-webhook"` | Display name in Linear webhook settings |
| `signal` | `AbortSignal?` | — | External shutdown signal |

**Returns:** `WebhookServer` with `handler`, `webhookId`, and `shutdown()`.

---

### useLinearWebhook

A React hook for use inside Smithers workflows. Starts a webhook server on mount, accumulates issues that match a trigger label, and cleans up on unmount.

```tsx
const { issues, ready, dismiss, shutdown } = useLinearWebhook({
  publicUrl: "https://my-tunnel.ngrok.io",
  triggerLabel: "smithers",  // fires when this label is newly added
  teamId: "team-uuid",
});

// issues: WebhookIssueEvent[] — triggered issues queue
// dismiss(issueId) — remove from queue after processing
```

**Options:**

| Option | Type | Default | Description |
|---|---|---|---|
| `publicUrl` | `string` | *required* | Public URL for webhook delivery |
| `triggerLabel` | `string` | *required* | Label name that triggers processing |
| `port` | `number` | `3456` | Local server port |
| `teamId` | `string?` | — | Scope to a specific team |
| `label` | `string` | `"smithers-webhook"` | Linear webhook display name |

**Returns:** `UseLinearWebhookResult` with `issues`, `ready`, `webhookId`, `dismiss()`, and `shutdown`.

The hook only fires for issues where the trigger label was **newly added** (not already present). Each issue is deduplicated by ID.

---

### LinearWebhookListener

A renderless React component that wraps `useLinearWebhook`. Use it inside JSX workflows to react to Linear events declaratively.

```tsx
<LinearWebhookListener
  publicUrl="https://my-tunnel.ngrok.io"
  triggerLabel="smithers"
  teamId="team-uuid"
  onIssue={(event) => {
    console.log(`New issue: ${event.identifier} — ${event.title}`);
  }}
  onReady={({ webhookId, shutdown }) => {
    console.log(`Webhook ${webhookId} ready`);
  }}
/>
```

**Props:** All `useLinearWebhook` options plus:

| Prop | Type | Description |
|---|---|---|
| `onIssue` | `(event: WebhookIssueEvent) => void` | Called for each triggered issue |
| `onReady` | `(info) => void` | Called when the webhook server is ready |

### WebhookIssueEvent

```ts
type WebhookIssueEvent = {
  action: string;
  issueId: string;
  identifier: string;    // e.g. "ENG-42"
  title: string;
  teamId: string;
  labelIds: string[];
  labels: { id: string; name: string; color: string }[];
  description: string | null;
  url: string;
  updatedFrom: Record<string, unknown> | undefined;
};
```

## getLinearClient

For advanced use cases, access the underlying `@linear/sdk` client directly:

```ts
import { getLinearClient } from "smithers-orchestrator/linear";

const client = getLinearClient();
const viewer = await client.viewer;
```

The client is a lazy singleton — it's created once on first access using `LINEAR_API_KEY` and cached for the process lifetime. Call `resetLinearClient()` to clear the cache (useful in tests):

```ts
import { resetLinearClient } from "smithers-orchestrator/linear";

resetLinearClient();
```

> `resetLinearClient` is only available from `smithers-orchestrator/linear`, not from the main barrel export.
