---
title: HTTP Server
description: Expose workflows over HTTP.
---

Smithers can run workflows via a built-in HTTP server.

```ts
import { startServer } from "smithers/server";
import { drizzle } from "drizzle-orm/bun-sqlite";

const db = drizzle("./smithers.db");

startServer({
  port: 7331,
  db,
  authToken: process.env.SMITHERS_API_KEY,
  rootDir: process.cwd(),
  allowNetwork: false,
});
```

**Server options**

| Field | Type | Description |
| --- | --- | --- |
| `port` | `number` | Port to listen on (default 7331) |
| `db` | `BunSQLiteDatabase` | Optional DB for run/event mirror |
| `authToken` | `string` | Bearer token or `SMITHERS_API_KEY` |
| `maxBodyBytes` | `number` | Max request body size |
| `rootDir` | `string` | Workflow root directory |
| `allowNetwork` | `boolean` | Allow network for `bash` tool |

**Endpoints**

- `POST /v1/runs` start a run
- `POST /v1/runs/:runId/resume` resume a run
- `POST /v1/runs/:runId/cancel` cancel a run
- `GET /v1/runs/:runId` run status summary
- `GET /v1/runs/:runId/frames?limit=N&afterFrameNo=M` list frames
- `GET /v1/runs/:runId/events` SSE event stream
- `POST /v1/runs/:runId/nodes/:nodeId/approve` approve a node
- `POST /v1/runs/:runId/nodes/:nodeId/deny` deny a node
- `GET /v1/runs?limit=N&status=STATUS` list runs (requires server DB)
