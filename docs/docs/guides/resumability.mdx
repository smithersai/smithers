---
title: Resumability
description: Resume interrupted runs safely and deterministically.
---

Smithers persists all run state to SQLite. When a process exits mid-run, you can resume using the same `runId` and Smithers will skip tasks that already wrote valid output rows.

**Key ideas**

- Node identity is deterministic: `nodeId` comes from `<Task id="...">`
- Outputs are keyed by `(runId, nodeId[, iteration])`
- If an output row exists and validates, the task is skipped

## Resume with the CLI

```bash
# start a run
smithers run workflow.tsx --run-id demo --input '{"description":"Fix bugs"}'

# resume after a crash or cancellation
smithers resume workflow.tsx --run-id demo
```

## Resume programmatically

```ts
import { runWorkflow } from "smithers";
import workflow from "./workflow";

await runWorkflow(workflow, {
  runId: "demo",
  input: { description: "Fix bugs" },
});

// ... later
await runWorkflow(workflow, {
  runId: "demo",
  resume: true,
  input: { description: "Fix bugs" },
});
```

Notes:

- On resume, the input row must already exist. Smithers will throw if it is missing.
- Use stable task ids. Changing ids changes node identity and forces re-execution.
