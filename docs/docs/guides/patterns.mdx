These patterns are distilled from a production Smithers workflow and are safe defaults for new projects.

**Project layout**

```
my-workflow/
  agents.ts
  db.ts
  config.ts
  workflow.tsx
  components/
    Discover.tsx
    Discover.schema.ts
    Implement.tsx
    Implement.schema.ts
    Report.tsx
    Report.schema.ts
  prompts/
    discover.mdx
    implement.mdx
  lib/
    render.ts
    ctx.ts
    coerce.ts
  bunfig.toml
  preload.ts
```

**Patterns to follow**

- Keep each task in its own component file and keep its table schema alongside it in `*.schema.ts`.
- Keep agent configuration in `agents.ts` and decide CLI vs API by env flags.
- Use MDX templates for prompts and render them to plain text.
- Keep shared constants in `config.ts`.
- Use helper functions for typed output access and JSON coercion.
- Use stable task ids and consistent naming like `${ticketId}:review`.

**Prompt rendering helper**

```ts
// lib/render.ts
import React from "react";
import { renderToStaticMarkup } from "react-dom/server";
import type { MDXContent } from "mdx/types";

export function render(Component: MDXContent, props: Record<string, any> = {}): string {
  const html = renderToStaticMarkup(React.createElement(Component, props));
  return html
    .replace(/<\/?(p|div|h[1-6]|li|ul|ol|pre|blockquote|section)[^>]*>/gi, "\n")
    .replace(/<br\s*\/?>(?!\n)/gi, "\n")
    .replace(/<[^>]+>/g, "")
    .replace(/&amp;/g, "&")
    .replace(/&lt;/g, "<")
    .replace(/&gt;/g, ">")
    .replace(/&quot;/g, '"')
    .replace(/&#x27;/g, "'")
    .replace(/&#x2F;/g, "/")
    .replace(/\n{3,}/g, "\n\n")
    .trim();
}
```

**Typed output helper**

```ts
// lib/ctx.ts
import type { SmithersCtx } from "smithers";
import type { SQLiteTable } from "drizzle-orm/sqlite-core";

export type WorkflowCtx = SmithersCtx<any>;

export function typedOutput<T>(ctx: WorkflowCtx, table: SQLiteTable, args: { nodeId: string }): T | undefined {
  const rows = (ctx.outputs as any)(table) as any[] | undefined;
  if (!rows?.length) return undefined;
  let latest: any;
  let latestIteration = -Infinity;
  for (const row of rows) {
    if (!row || row.nodeId !== args.nodeId) continue;
    const iter = Number.isFinite(Number(row.iteration)) ? Number(row.iteration) : 0;
    if (!latest || iter >= latestIteration) {
      latest = row;
      latestIteration = iter;
    }
  }
  return latest as T | undefined;
}
```

**Schema placement**

Keep table definitions and Zod schema together. Smithers validates against the Drizzle table schema; Zod is still useful for prompt templates and JSON examples.

```ts
// components/Discover.schema.ts
import { z } from "zod";
import { sqliteTable, text, integer, primaryKey } from "drizzle-orm/sqlite-core";

export const discoverTable = sqliteTable(
  "discover",
  {
    runId: text("run_id").notNull(),
    nodeId: text("node_id").notNull(),
    iteration: integer("iteration").notNull().default(0),
    tickets: text("tickets", { mode: "json" }).$type<any[]>().notNull(),
  },
  (t) => ({ pk: primaryKey({ columns: [t.runId, t.nodeId, t.iteration] }) }),
);

export const ticketSchema = z.object({
  id: z.string(),
  title: z.string(),
  description: z.string(),
});

export const discoverOutputSchema = z.object({
  tickets: z.array(ticketSchema),
});
```

**Task ids and `skipIf`**

`skipIf` is evaluated during render. If it is `true`, the task is marked `skipped` and will not run later even if the condition changes. For tasks that should run after prerequisites, prefer `Sequence` and use `ctx.outputMaybe(...)` instead of `skipIf`.
