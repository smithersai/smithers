---
title: Monitoring & Logs
description: Inspect progress with events and NDJSON logs.
---

Smithers emits `SmithersEvent` objects for every run. Events are persisted to SQLite and optionally written as NDJSON to a log file.

**Default log path**

```
.smithers/
└── executions/<runId>/logs/
    └── stream.ndjson
```

You can change or disable log output:

```bash
smithers run workflow.tsx --log-dir .smithers/logs
smithers run workflow.tsx --no-log
```

## NDJSON stream

Each line is a JSON-encoded `SmithersEvent`.

```json
{"type":"RunStarted","runId":"abc","timestampMs":1730000000000}
{"type":"NodeStarted","runId":"abc","nodeId":"analyze","iteration":0,"attempt":1,"timestampMs":1730000000100}
```

## Live monitoring

- CLI: `smithers status workflow.tsx --run-id <id>`
- Server: `GET /v1/runs/:runId/events` (SSE)

## Database events

Events are stored in `_smithers_events` with `payload_json` per event. You can query with SQLite:

```bash
sqlite3 workflow.db "SELECT seq, type, payload_json FROM _smithers_events WHERE run_id = 'abc' ORDER BY seq DESC LIMIT 20;"
```
